# **4. UI/UX 화면 설계 및 가이드 (UI/UX Design & Guide)**

## **1. 개요 (Overview)**

본 시스템은 **Bootstrap 5.3** 기반의 반응형 웹 인터페이스를 제공하며, 복잡한 해운 물류 데이터를 직관적이고 효율적으로 편집할 수 있도록 설계되었습니다.

특히 대용량 데이터의 원활한 처리를 위해 **DataTables Server-side Rendering (SSR)**을 도입하였고, 코드 중복을 최소화하기 위해 템플릿 상속과 **공통 UI 팩토리 패턴**을 적용했습니다.

### **1.1 UI 표준 (UI Standards)**

| 요소 | 기술 | 버전 | 용도 |
|------|------|------|------|
| **CSS Framework** | Bootstrap | 5.3 | 반응형 레이아웃, 컴포넌트 기본 스타일 |
| **Icons** | Bootstrap Icons | 1.11.0 | 내비게이션, 버튼 아이콘 |
| **Data Table** | DataTables | 2.0 | Server-side Pagination, Filtering, Sorting |
| **Layout** | base.html (Template Inheritance) | Django | 전체 페이지 레이아웃 및 사이드바 |
| **Navigation** | common/menus.py (Dynamic Menu) | Python Dict | 2-Depth 동적 메뉴 렌더링 |
| **Notifications** | messages_display.html (Django Messages) | Toast/Alert | 화면 상단 피드백 메시지 |
| **JavaScript** | common_api.js (Utility Module) | Fetch API | 비동기 AJAX 호출, 종속성 제어 |

### **1.2 화면 아키텍처 개요**

```
base.html (최상위 부모)
│
├── 사이드바 (Sidebar Navigation)
│   └── MENU_STRUCTURE (creation, input_management, master)
│       ├── [Creation] Scenario, Proforma, Cascading, LRS, ...
│       ├── [Input Management] Cost, Bunker, Vessel, ...
```

## **2. 공통 UI 템플릿 아키텍처 (Common UI Architecture)**

단순 반복되는 CRUD(목록 조회, 추가, 삭제) 화면의 유지보수성을 극대화하기 위해, 프론트엔드(_datatables_base.html)와 백엔드(_crud_base.py)가 완벽하게 짝을 이루는 **공통 팩토리 아키텍처**를 구축했습니다. 용도에 따라 두 가지 베이스 템플릿으로 분리되어 운영됩니다.

### **2.1 투-트랙(Two-Track) 베이스 템플릿 (핵심 공통 뼈대)**

#### **Track 1: _datatables_base.html (시나리오 데이터 전용)**
* 비용(Cost), 선박(Vessel), 연료(Bunker) 등 특정 시나리오(scenario_id)에 종속된 파생 데이터 화면이 상속받습니다.
* 상단에 항상 **Scenario 선택 Select Box**가 강제되며, 선택된 시나리오에 맞는 데이터만 AJAX로 불러옵니다.
* 상속 예시: `cost_list.html`, `bunker_list.html`, `vessel_list.html` 등

#### **Track 2: master_list_base.html (마스터 데이터 전용)**
* Trade, Port, Lane, Week Period 등 시나리오와 무관한 전사 공통 불변 데이터 화면이 상속받습니다.
* 시나리오 선택 영역이 없으며, {% if has_csv %} 블록을 통해 마스터 데이터 전체에 대한 CSV 다운로드/업로드 기능을 동적으로 제공합니다.
* 상속 예시: `master_trade_list.html`, `master_port_list.html`, `master_lane_list.html`

#### **[공통 제공 기능]**

**자동화된 Server-side DataTables 연동:**
* 대용량 데이터 렌더링 성능을 위해 AJAX 기반의 서버사이드 프로세싱 로직이 기본으로 내장
* 페이징, 정렬, 검색 이벤트 발생 시 현재 URL로 자동 비동기 요청
* 백엔드 팩토리가 반환하는 JSON 데이터를 파싱하여 화면을 즉각 갱신
* 흐름: 사용자 입력 → JavaScript 이벤트 → AJAX 요청 → Backend JSON 응답 → DOM 갱신

**표준 레이아웃 및 컴포넌트 통합:**
* **액션 버튼 및 폼 구조:**
  - Add Row 버튼: 신규 데이터 추가 모달 팝업
  - Delete Selected 버튼: 선택된 행들 일괄 삭제 (확인 알럿)
  - CSV Download 버튼: 필터 상태 그대로 CSV 내보내기
  - CSV Upload 버튼: Excel/CSV 파일 대량 일괄 로드
  - 모든 폼은 CSRF 토큰 포함 ({% csrf_token %})

**Block Overriding (자식 템플릿의 주입 방식):**
자식 템플릿(예: canal_fee_list.html)은 전체 HTML 레이아웃을 신경 쓸 필요 없이, 다음 블록들로 필요한 내용을 주입합니다:

```django
{% block extra_filters %}
  <!-- 도메인 특화 검색 필터 (예: Port Code, Vessel Code) -->
{% endblock %}

{% block table_headers %}
  <!-- DataTables 헤더 UI -->
{% endblock %}

{% block dt_columns %}
  <!-- DataTables JSON 매핑 -->
{% endblock %}

{% block modal_inputs %}
  <!-- 'Add Row' 모달 창 내부 입력 폼 -->
{% endblock %}
```

### **2.2 클라이언트 사이드 확장 훅 (JavaScript Hooks)**

부모 템플릿에서 선언된 DataTables 객체와 모달 제어 로직을 자식 템플릿에서 유연하게 다룰 수 있도록 글로벌 함수 훅(Hook) 패턴을 제공합니다.

```javascript
// 1. DataTables AJAX 요청 시 커스텀 파라미터 추가
window.appendDtParams = function(d) {
  // 자식 템플릿에서 정의된 필터 값 추가
};

// 2. 모달 열릴 때마다 이전 입력값 초기화
window.resetModalForm = function() {
  // 모든 input 필드를 공백으로 리셋
};

// 3. DataTables 인스턴스에 부가 이벤트 바인딩
window.bindExtraEvents = function(table) {
  // 엔터 키 입력 시 검색 트리거 등
};
```

### **2.3 데이터 조작 워크플로우 (Data Manipulation Flow)**

#### **모달 기반 데이터 추가 (Add Row)**
```
[Add Row 버튼 클릭]
    ↓
[Bootstrap Modal 열기]
    ↓
[window.resetModalForm() 호출 - 이전 입력값 초기화]
    ↓
[사용자가 입력 필드 채우기]
    ↓
[Save 버튼 클릭]
    ↓
[JavaScript: new_필드명_0 네이밍 규칙으로 폼 데이터 수집]
    ↓
[AJAX POST 요청]
    ↓
[백엔드 팩토리 뷰: update_or_create() 실행]
    ↓
[Modal 닫기 + Success 메시지]
    ↓
[DataTables.draw() 호출 - 테이블 자동 갱신]
```

#### **체크박스 일괄 삭제 (Bulk Delete)**
```
[각 행의 체크박스 선택]
    ↓
[Delete Selected 버튼 클릭]
    ↓
[확인 알럿(Alert) 팝업]
    ↓
[사용자가 'Yes' 선택]
    ↓
[action="delete" 플래그 + selected_pks POST 전송]
    ↓
[백엔드: 다건 삭제 실행]
    ↓
[Warning 메시지 + DataTables.draw()]
```

## **3. 화면별 상세 설계 (Screen Specifications)**

### **3.1 대시보드 (Dashboard / input_home.html)**

시스템 접속 시 가장 먼저 표시되는 메인 요약 화면입니다.

**주요 기능:**
- **Status Cards**: 전체 시나리오 개수, 오늘 생성된 시나리오, 마지막 업데이트 시간
- **Create Base Scenario 버튼**: Fetch API로 비동기 호출, 로딩 스피너 표시
- **Recent Scenarios 테이블**: 최근 수정된 시나리오 5개

### **3.2 시나리오 관리 화면 (scenario_list.html & scenario_dashboard.html)**

#### **시나리오 목록 (scenario_list.html)**
- Server-side DataTables로 대용량 시나리오 처리
- New Scenario 드롭다운: Create Empty / Clone from...
- Delete 시 강력한 경고 모달 표시

#### **시나리오 대시보드 (scenario_dashboard.html)**
- **KPI 카드**: Total Lanes, Required Vessels, Total Capacity (TEU)
- **Lane-wise Deployment Plan 테이블**: 노선별 Configuration 현황
- 색상 코딩: ✓ (완성), ✗ (미완성)

### **3.3 Proforma Schedule 편집 (proforma_create.html)**

선박 운항 스케줄을 엑셀처럼 편집하는 핵심 화면입니다.

**주요 기능:**
1. **Dynamic Grid UI**: 엑셀 유사 인터페이스, Tab/Shift+Tab 키보드 내비게이션
2. **Read-only Cells**: 자동 계산 컬럼 (Distance, Sea Time, ETD) 회색 처리
3. **자동 역산 엔진**: Distance → Sea Time → Speed → ETD 자동 계산
4. **ETB 역전 방어**: Red flash 효과 + 자동 보정 (다음 주차 동일 요일로 조정)
5. **액션 버튼**: New, Load, Calculate, Save, Download, Upload

**계산 순서 예시:**
```
Port A (ETB: 2026-01-01 09:00)
  ↓ [Work: 36h]
  ETD: 2026-01-02 21:00
  ↓ [Distance: 720nm, Speed: 20kn → Sea Time: 36h]
  
Port B (ETB: 2026-01-04 09:00)
```

### **3.4 Lane Proforma Mapping (lane_proforma_mapping.html)**

노선(Lane)에 시기별로 적용할 스케줄(Proforma)을 매핑하는 화면입니다.

**UI 형태 - 가로형 타임라인 바:**
- 색상 코딩: Blue (Active), Light Blue (In-range), Gray (Inactive)
- 드래그로 기간 조정, Date Picker로 정확한 날짜 입력
- Save Changes 버튼으로 변경사항 저장

### **3.5 Cascading UI**

#### **Cascading Schedule 생성 (cascading_create.html)**
- Position 셀을 클릭(toggleCell)하여 토글 선택
- 선택된 Position: 파란색 배경 + "Selected" 텍스트
- Hidden Input 자동 생성 (hiddenSlotsContainer)

#### **Cascading Vessel 배치 (cascading_vessel_create.html)**
- Position별 Vessel Code, Own/Charter, 투입일 지정
- Auto-fill: Vessel Code 선택 시 vessel_name 자동 채우기
- 투입일 자동 계산: Position 1 + Duration × (Position - 1)
- Save & Generate LRS로 완료 후 LRS 조회 페이지로 이동

#### **Cascading Schedule 정보 (cascading_schedule_list.html)**
- 노선/Proforma별 Configuration 현황 (4/4 ✓)
- 행 클릭 → 편집 모드 진입
- 진행률 표시

#### **Cascading Vessel 정보 (cascading_vessel_info.html)**
- 달력 형태로 선박 배치 현황 시각화
- 색상: 초록색 (배치됨) ●, 회색 (미배치) ○
- Tooltip: 마우스 오버 시 선박 코드 표시

### **3.6 Long Range Schedule (LRS) 조회 (long_range_list.html)**

**4단계 연쇄 필터 (Cascade Select):**
```
Scenario 선택 → AJAX로 Lane 옵션 로드
Lane 선택 → Proforma 옵션 로드
Proforma 선택 → Vessel 옵션 로드
```

**대용량 데이터 처리:**
- Server-side DataTables (페이징, 정렬, 검색)
- 5,000+ 레코드도 빠른 응답
- Virtual Port 시각화 (파란 텍스트 + "[Virtual]" 라벨)

### **3.7 마스터 및 기준 데이터 화면**

#### **Master Data (master_*.html)**
- master_list_base.html 상속
- CSV 다운로드/업로드 기능 통합
- 시나리오 뎁스 제거 (공통 기준 데이터)

#### **기준 데이터 (cost_list.html, bunker_list.html, vessel_list.html)**
- _datatables_base.html 상속
- 상단 Scenario Select Box 강제

## **4. 프론트엔드 자바스크립트 모듈 (common_api.js)**

### **4.1 주요 함수들**

#### **연쇄 로드 함수 (Cascade Loading)**
```javascript
// Lane 로드
async function loadLanes(scenarioId) {
  const response = await fetch(`/api/scenarios/${scenarioId}/lanes/`);
  const data = await response.json();
  populateSelect('#laneSelect', data, 'lane_code', 'lane_name');

// Proforma 로드
async function loadProformas(scenarioId, laneCode) {
  const response = await fetch(

  const data = await response.json();

// Vessel 옵션 로드
async function loadVesselOptions(scenarioId, laneCode, proformaName) {
```


#### **에러 처리 함수**

```javascript
  if (error instanceof TypeError) {
    showAlert('Network connection failed', 'danger');
  } else {
    showAlert('Error: ' + error.status, 'danger');
  }

```

### **4.2 유틸리티 함수**

```javascript
// Select Box 채우기
function populateSelect(selectId, data, valueField, textField) {
  const select = document.querySelector(selectId);
  select.innerHTML = '<option value="">-- Select --</option>';
  data.forEach(item => {
    const option = document.createElement('option');
    option.value = item[valueField];
    option.textContent = item[textField];
    select.appendChild(option);
  });
  select.disabled = false;
}

// 체크박스 토글 (Cascading UI용)
function toggleCell(cell) {
  cell.classList.toggle('selected');
  updateHiddenSlots();
}

// Hidden Input 업데이트
function updateHiddenSlots() {
  const selected = Array.from(document.querySelectorAll('.cell.selected'))
    .map(cell => cell.dataset.position_id);
  document.getElementById('hiddenSlotsInput').value = selected.join(',');
}
```

## **5. 반응형 디자인 및 접근성 (Responsive & Accessibility)**

### **5.1 반응형 브레이크포인트**

| 디바이스 | 너비 | 적용 사항 |
|---------|------|---------|
| **Mobile** | < 768px | 사이드바 숨김, 햄버거 메뉴, 단일 컬럼 |
| **Tablet** | 768px - 1024px | 사이드바 축소, 2컬럼 레이아웃 |
| **Desktop** | ≥ 1025px | 전체 레이아웃 (사이드바 + 콘텐츠) |

### **5.2 접근성 (Accessibility)**

- **ARIA 라벨**: 모든 버튼, 입력 필드에 적절한 `aria-label` 추가
- **키보드 내비게이션**: Tab 키로 모든 대화형 요소 접근 가능
- **스크린 리더**: 시맨틱 HTML, 대체 텍스트 포함
