# **4\. UI/UX 화면 설계 및 가이드 (UI/UX Design & Guide)**

## **1\. 개요 (Overview)**

본 시스템은 **Bootstrap 5.3** 기반의 반응형 웹 인터페이스를 제공하며, 복잡한 해운 물류 데이터를 직관적이고 효율적으로 편집할 수 있도록 설계되었습니다.

특히 대용량 데이터의 원활한 처리를 위해 \*\*DataTables Server-side Rendering (SSR)\*\*을 도입하였고, 코드 중복을 최소화하기 위해 템플릿 상속과 **공통 UI 팩토리 패턴**을 적용했습니다.

### **1.1 UI 표준 (UI Standards)**

* **Framework:** Bootstrap 5.3 (CSS/JS) \+ Bootstrap Icons  
* **Grid & Table:** DataTables 2.0 (AJAX Server-side Processing)  
* **Layout:** base.html을 최상위 부모로 상속받는 구조  
* **Sidebar:** common/menus.py의 MENU\_STRUCTURE 딕셔너리에 기반한 2-Depth 동적 렌더링  
* **Notifications:** messages\_display.html을 통해 화면 상단에 Toast / Alert 형태로 피드백 제공

## **2\. 공통 UI 템플릿 아키텍처 (Common UI Architecture)**

단순 반복되는 CRUD(목록 조회, 추가, 삭제) 화면의 유지보수성을 극대화하기 위해, 프론트엔드(\_datatables\_base.html)와 백엔드(\_crud\_base.py)가 완벽하게 짝을 이루는 **공통 팩토리 아키텍처**를 구축했습니다. 이를 통해 새로운 데이터 관리 화면을 추가할 때 프론트엔드 코드를 80% 이상 절감할 수 있습니다.

### **2.1 \_datatables\_base.html (핵심 공통 뼈대)**

비용(Cost), 선박(Vessel), 연료(Bunker) 등 대부분의 마스터 및 시나리오 데이터 화면이 이 최상위 템플릿을 상속받아 구현됩니다.

* **자동화된 Server-side DataTables 연동:** \* 대용량 데이터 렌더링 성능을 위해 AJAX 기반의 서버사이드 프로세싱 로직이 기본으로 내장되어 있습니다.  
  * 페이징, 정렬, 검색 이벤트 발생 시 현재 URL로 자동 비동기 요청을 보내고, 백엔드 팩토리가 반환하는 JSON 데이터를 파싱하여 화면을 즉각 갱신합니다.  
* **표준 레이아웃 및 컴포넌트 통합:** \* **검색 필터 영역:** 시나리오 선택 드롭다운, 기준 연월(Base Year Month) 등 모든 화면에서 쓰이는 공통 검색 조건을 기본 제공합니다.  
  * **액션 버튼 및 폼 구조:** 데이터 추가(Add Row), 선택 삭제(Delete Selected), CSV 일괄 업로드/다운로드 버튼이 폼(deleteForm, addForm) 구조 안에 미리 설계되어 있어 안전한 POST 요청을 보장합니다.  
* **Block Overriding (자식 템플릿의 주입 방식):**  
  자식 템플릿(예: canal\_fee\_list.html)은 전체 HTML 레이아웃을 신경 쓸 필요 없이, 자신에게 필요한 알맹이만 아래의 블록({% block %})으로 주입하여 화면을 완성합니다.  
  * extra\_filters: 도메인 특화 커스텀 검색 조건 (예: Port Code, Vessel Code 등)  
  * table\_headers / dt\_columns: DataTables의 헤더 UI와 서버 JSON 데이터를 매핑하는 설정  
  * modal\_inputs: 'Add Row' 클릭 시 팝업되는 모달 창 내부의 개별 데이터 입력 폼

### **2.2 클라이언트 사이드 확장 훅 (JavaScript Hooks)**

부모 템플릿에서 선언된 DataTables 객체와 모달 제어 로직을 자식 템플릿에서 유연하게 다룰 수 있도록 글로벌 함수 훅(Hook) 패턴을 제공합니다.

* window.appendDtParams(d): DataTables의 AJAX 서버 요청을 보낼 때, 자식 템플릿에 정의된 커스텀 검색 필터 값들을 파라미터(d)에 동적으로 추가합니다.  
* window.resetModalForm(): 모달 창이 새로 열릴 때마다 이전 입력값들을 초기화하는 도메인별 맞춤 로직을 구현합니다.  
* window.bindExtraEvents(table): 특정 Input 필드에서 엔터 키를 누르면 검색이 트리거되게 하는 등, DataTables 인스턴스에 부가적인 이벤트를 바인딩할 때 사용합니다.

### **2.3 데이터 조작 워크플로우 (Data Manipulation Flow)**

* **모달 기반 데이터 추가 (Add Row):** 새로운 레코드 생성 시 화면 이동 없이 Bootstrap 모달이 열립니다. 자식 템플릿에 정의된 입력 필드들은 new\_필드명\_0의 네이밍 규칙을 따르며, 백엔드의 팩토리 뷰가 이를 자동 파싱하여 중복 체크 후 DB에 저장(Upsert/Create)합니다.  
* **체크박스 일괄 삭제 (Bulk Delete):** 테이블의 각 행에 바인딩된 체크박스(selected\_pks)를 선택하고 삭제 버튼을 누르면, 자바스크립트가 확인 알럿(Alert)을 띄운 후 action="delete" 플래그와 함께 POST 폼을 전송하여 다건 삭제를 한 번에 처리합니다.

## **3\. 화면별 상세 설계 (Screen Specifications)**

### **3.1 대시보드 (Dashboard / input\_home.html)**

시스템 접속 시 가장 먼저 표시되는 메인 요약 화면입니다.

* **Status Card:** 전체 시나리오 개수, 최근 업데이트 일자 등의 요약 정보 제공.  
* **Base Scenario Action:** "Create Base Scenario" 버튼 클릭 시, 페이지 이동 없이 비동기(Fetch API)로 백엔드를 호출하여 마스터 데이터로부터 즉시 초기 시나리오를 덤프합니다 (버튼 내 로딩 스피너 적용).

### **3.2 시나리오 대시보드 (scenario\_dashboard.html)**

특정 시나리오의 상세 정보를 한눈에 파악할 수 있는 개별 대시보드입니다.

* **KPI 시각화:** 해당 시나리오의 Total Lanes, Required Vessels, Total Capacity(TEU), Simulation Runs 횟수 등을 상단 카드 형태로 제공합니다.  
* **요약 테이블:** 노선별 투입된 Proforma 이름과 선박 수, 선박 크기, 시작일 등을 요약하여 보여주며, 바로 수정(Edit) 화면으로 넘어갈 수 있는 Action 링크를 제공합니다.

### **3.3 Proforma Schedule 편집 (proforma\_create.html)**

선박 운항 스케줄을 엑셀처럼 편집하는 핵심 화면입니다.

* **Dynamic Grid UI:** 엑셀 시트와 유사하게 디자인되었으며, JS를 통해 행(Row) 추가/삽입/삭제가 즉각적으로 이루어집니다.  
* **Read-only Cells 방어:** 백엔드에서 자동 계산되는 컬럼(Distance, Sea Time, ETD 등)은 cell-readonly 클래스를 적용해 회색 배경으로 처리하고, 사용자의 입력을 물리적/시각적으로 차단하여 실수를 방지합니다.  
* **ETB 역전 방어 로직:** 입력된 접안(ETB) 시간이 이전 항구의 출항(ETD) 시간보다 과거일 경우, JS 단에서 즉각적인 경고(Alert) 후 이전 ETD 값으로 롤백시키는 시각적 효과(Red flash)를 주어 데이터 정합성을 보호합니다.

### **3.4 Lane Proforma Mapping (lane\_proforma\_mapping.html)**

노선(Lane)에 시기별로 적용할 스케줄(Proforma)을 매핑하는 화면입니다.

* **가로형 타임라인 바 (Timeline Bar):** 복잡한 주차별(Year-Week) 매핑 상태를 달력 형태의 가로 스크롤 테이블로 시각화했습니다.  
* **색상 코딩(Color Coding):**  
  * Blue (Active): 체크박스로 선택되었으며 실제 적용 중인 기간  
  * Light Blue (In-range): 해당 Proforma의 유효 기간(Effective Date) 내에 있으나 아직 선택되지 않은 기간  
  * Gray (Inactive): 유효 기간 밖

### **3.5 선박 배선 대시보드 (Cascading UI)**

* **cascading\_vessel\_info.html**: 노선/Proforma별로 선박이 어떻게 배치되었는지 한눈에 봅니다. 선박이 배치된 슬롯은 초록색, 비어있는 슬롯은 회색 점으로 표시되며, 마우스를 올리면(Tooltip) 선박 코드를 확인할 수 있습니다.  
* **cascading\_create.html**: 사용자가 직접 슬롯을 클릭하여 토글(Select/Deselect) 형태로 직관적으로 배선 대상 슬롯을 활성화/비활성화합니다.

### **3.6 Vessel Info 마스터/시나리오 연동 (vessel\_info\_list.html)**

* **Auto-fill (자동 채움) UI:** 모달 폼에서 BaseVesselInfo에 등록된 마스터 선박 코드를 Select Box로 선택하면, HTML dataset 속성을 활용해 선박명(vessel\_name)과 소유/용선 여부(own\_yn)를 다른 Input 필드에 자동으로 채워주어 사용자의 입력 피로도를 낮춥니다.

## **4\. 프론트엔드 자바스크립트 모듈 (common\_api.js)**

연쇄적인 데이터 필터링(Scenario \-\> Lane \-\> Proforma \-\> Vessel)을 깔끔하게 처리하기 위한 공통 유틸리티입니다.

* **CommonApi.populateSelect**: 백엔드 API로부터 받아온 JSON 배열을 파싱하여 타겟 \<select\> 태그의 \<option\>을 동적으로 채워주고 활성화합니다.  
* **Error Handling**: \_handleFetchResponse 헬퍼 함수를 통해 HTTP 에러를 낚아채어 UI가 무한 로딩에 빠지지 않도록 방어합니다.
