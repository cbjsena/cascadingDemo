# **5. 개발 환경 및 운영 매뉴얼 (Development & Operations Manual)**

## **1. 개요 (Overview)**

본 문서는 Cascading Opt 시스템의 로컬 개발 환경 구성(Local Setup), Docker 기반 실행, 초기 데이터 적재 자동화, 테스트 환경, 운영 모니터링 및 트러블슈팅 가이드를 제공합니다.

### **1.1 시스템 요구사항 (Prerequisites)**

| 항목 | 요구사항 | 비고 |
|------|---------|------|
| **OS** | Windows 10/11 (WSL2 권장), macOS, Linux | WSL2 환경에서 Linux처럼 동작 |
| **Language** | Python 3.11+ | Django 5.2 호환성 |
| **Database** | PostgreSQL 14+ | 로컬 테스트 및 운영 환경 필수 |
| **Cache/Queue** | Redis 7+ | Celery 비동기 작업용 메시지 브로커 |
| **Containerization** | Docker Desktop, Docker Compose | 또는 Podman |
| **VCS** | Git | GitHub, GitLab, Gitea 등 |
| **IDE** | PyCharm, VS Code, Vim | Python 3.11 인터프리터 설정 필요 |

## **2. 신규 개발자 온보딩 체크리스트 (Quick Start - 30분)**

처음 프로젝트를 시작하는 개발자를 위한 단계별 가이드입니다.

### **Step 1. 저장소 클론 (5분)**
```bash
# 1. 저장소 클론
git clone <repository_url>
cd cascadingDemo

# 2. 브랜치 확인 (develop 또는 main)
git branch -a
git checkout develop  # 또는 main
```

### **Step 2. Python 가상환경 설정 (10분)**
```bash
# 1. 가상환경 생성 (Python 3.11 필수)
python3.11 -m venv venv

# 2. 가상환경 활성화
# Windows:
venv\Scripts\activate
# macOS/Linux:
source venv/bin/activate

# 3. pip 업그레이드
pip install --upgrade pip setuptools wheel

# 4. 의존성 패키지 설치
pip install -r requirements.txt

# 5. Pre-commit Hook 설치 (코드 품질 자동 관리)
pre-commit install

# 6. 설치 확인
python --version  # Python 3.11.x 확인
django-admin --version  # Django 5.2.x 확인
```

### **Step 3. 환경 변수 설정 (5분)**
```bash
# 1. .env.local 파일 생성 (프로젝트 루트)
cp .env.example .env.local

# 2. 에디터로 열어 다음 내용 수정:
# DATABASE_URL=postgresql://user:password@localhost:5432/cascading_opt
# DEBUG=True
# SECRET_KEY=<Django secret key>
# CELERY_BROKER_URL=redis://localhost:6379/0

# 3. 파일 저장
```

### **Step 4. 데이터베이스 마이그레이션 (5분)**
```bash
# 1. 마이그레이션 파일 생성 (필요 시)
python manage.py makemigrations

# 2. 마이그레이션 실행
python manage.py migrate

# 자동으로 다음이 수행됨:
# - DB 메타데이터 동기화 (테이블 코멘트)
# - 테이블 정의서 생성 (doc/db/base_table_definitions.csv)
# - 슈퍼유저 자동 생성 (cascading / qwer123$)
# - 마스터 데이터 적재 (input_data/data/base_data/*.csv)
```

### **Step 5. 테스트 실행 (10분)**
```bash
# 1. 테스트 데이터베이스 생성 (자동)
# 2. 테스트 실행
pytest tests/ -v --tb=short

# 결과: 약 190개 테스트, 예상 실행 시간 2-3분
```

### **Step 6. 개발 서버 시작 (2분)**
```bash
# 1. 개발 서버 실행
python manage.py runserver

# 2. 브라우저 접속
# URL: http://localhost:8000
# 로그인: cascading / qwer123$
```

### **Step 7. 체크리스트 확인**
- ✅ Python 3.11 설치 확인
- ✅ 가상환경 활성화 (프롬프트에 (venv) 표시)
- ✅ requirements.txt 설치 완료
- ✅ .env.local 파일 생성 및 설정
- ✅ PostgreSQL 또는 Docker에서 DB 실행 중
- ✅ 마이그레이션 성공 (메시지: "Ran X migrations")
- ✅ 테스트 통과 (passed, 0 failed)
- ✅ 서버 실행 (메시지: "Starting development server at http://127.0.0.1:8000")

## **3. 환경 변수 설정 (.env)**

### **3.1 필수 환경변수 (Required)**

| 환경변수 | 값 예시 | 설명 |
|---------|--------|------|
| `DATABASE_URL` | `postgresql://user:pwd@localhost:5432/cascading_opt` | PostgreSQL 연결 문자열 |
| `DEBUG` | `True` (개발) / `False` (운영) | 디버그 모드 활성화 |
| `SECRET_KEY` | `django-insecure-xxxxx...` | Django 비밀키 (배포 시 변경 필수) |
| `CELERY_BROKER_URL` | `redis://localhost:6379/0` | Celery 메시지 브로커 |
| `APP_ENV` | `local` (개발) / `docker` (Docker) | 환경 식별자 |

### **3.2 옵션 환경변수 (Optional)**

| 환경변수 | 기본값 | 설명 |
|---------|-------|------|
| `ALLOWED_HOSTS` | `localhost,127.0.0.1` | 허용된 호스트 (`,` 로 구분) |
| `TIME_ZONE` | `Asia/Seoul` | 시스템 타임존 |
| `LANGUAGE_CODE` | `ko-kr` | 언어 설정 |
| `CELERY_TASK_ALWAYS_EAGER` | `False` | True로 설정 시 비동기 작업을 동기로 실행 (디버깅용) |

### **3.3 .env.local 예시 (로컬 개발)**
```env
# Database
DATABASE_URL=postgresql://postgres:password@localhost:5432/cascading_opt_dev

# Django
DEBUG=True
SECRET_KEY=django-insecure-dev-key-only-for-local-testing
ALLOWED_HOSTS=localhost,127.0.0.1
TIME_ZONE=Asia/Seoul
LANGUAGE_CODE=ko-kr

# Celery & Redis
CELERY_BROKER_URL=redis://localhost:6379/0
CELERY_TASK_ALWAYS_EAGER=False  # 또는 True (비동기 작업을 동기로 실행하여 디버깅)

# Environment
APP_ENV=local
```

## **5. Docker 기반 실행 (Docker Deployment)**

### **5.1 docker-compose.yml 서비스 구조**

```yaml
services:
  # 1. PostgreSQL 데이터베이스
  db:
    image: postgres:14-alpine
    environment:
      POSTGRES_PASSWORD: postgres
      POSTGRES_DB: cascading_opt
    ports:
      - "5432:5432"  # 로컬 포트:컨테이너 포트
    volumes:
      - postgres_data:/var/lib/postgresql/data

  # 2. Redis 메시지 브로커 (Celery용)
  redis:
    image: redis:7-alpine
    ports:
      - "6379:6379"
    volumes:
      - redis_data:/data

  # 3. Django 애플리케이션 서버
  web:
    build: .
    command: python manage.py runserver 0.0.0.0:8000
    ports:
      - "8000:8000"
    environment:
      - APP_ENV=docker
      - DATABASE_URL=postgresql://postgres:postgres@db:5432/cascading_opt
      - CELERY_BROKER_URL=redis://redis:6379/0
    depends_on:
      - db
      - redis
    volumes:
      - .:/code  # 코드 변경 시 자동 반영

  # 4. Celery 워커 (비동기 작업 처리)
  worker:
    build: .
    command: celery -A config worker -l info
    environment:
      - APP_ENV=docker
      - DATABASE_URL=postgresql://postgres:postgres@db:5432/cascading_opt
      - CELERY_BROKER_URL=redis://redis:6379/0
    depends_on:
      - db
      - redis
```

### **5.2 Docker 실행 명령어**

```bash
# 1. 컨테이너 빌드 및 시작 (백그라운드)
docker-compose up -d --build

# 2. 실행 상태 확인
docker-compose ps

# 3. 웹 컨테이너 내에서 마이그레이션 실행
docker-compose exec web python manage.py migrate

# 4. 로그 보기
docker-compose logs -f web    # Django 로그
docker-compose logs -f worker # Celery 로그

# 5. 컨테이너 중지
docker-compose down

# 6. 데이터 초기화 (주의: 모든 데이터 삭제)
docker-compose down -v  # -v는 named volumes도 제거
```

### **5.3 Docker 네트워크 및 포트**

| 서비스 | 컨테이너 포트 | 로컬 포트 | 접속 URL |
|--------|--------------|----------|----------|
| **Web** | 8000 | 8000 | http://localhost:8000 |
| **DB** | 5432 | 5432 | postgresql://localhost:5432 |
| **Redis** | 6379 | 6379 | redis://localhost:6379 |

## **6. 초기 데이터 적재 및 자동화 (Auto-Setup)**

### **6.1 post_migrate 자동화 순서도**

```
[python manage.py migrate 실행]
                ↓
[1. DB 메타데이터 동기화 (update_db_comments)] ⏱ ~5초
    └─ Django verbose_name → PostgreSQL COMMENT ON COLUMN
                ↓
[2. 테이블 정의서 생성 (generate_table_definition)] ⏱ ~2초
    └─ doc/db/base_table_definitions.csv 자동 갱신
                ↓
[3. 슈퍼유저 자동 생성 (create_default_superuser)] ⏱ ~1초
    └─ cascading / qwer123$ (기존 사용자 있으면 스킵)
                ↓
[4. 마스터 데이터 적재 (init_master_data)] ⏱ ~10초
    └─ input_data/data/master_data/*.csv
       └─ MasterTrade, MasterPort, MasterLane 등
                ↓
[5. 기준 데이터 적재 (init_base_data)] ⏱ ~30초
    └─ input_data/data/base_data/*.csv
       └─ BaseVesselInfo, BaseCost, BaseBunker 등
       └─ bulk_create(batch_size=5000)로 고속 처리
                ↓
[마이그레이션 완료] (총 소요 시간: ~50초)
```

### **6.2 수동 데이터 재적재**

```bash
# 1. 기준 데이터만 갱신 (시나리오 데이터는 유지)
python manage.py init_base_data

# 2. 마스터 데이터 갱신 (주의: 시나리오 데이터 연쇄 삭제!)
python manage.py init_master_data
# 경고 메시지 확인 후 yes 입력
# 또는 --force 옵션으로 자동 진행
python manage.py init_master_data --force

# 3. 특정 CSV만 로드 (향후 구현 가능)
# python manage.py init_base_data --table=vessel_info
```

### **6.3 적재 실패 시 복구**

```bash
# 1. 문제 파악: 로그 확인
python manage.py migrate --verbosity=3

# 2. 마이그레이션 파일 초기화 (주의!)
rm input_data/migrations/*.py  # 0001_initial.py 제외
rm common/migrations/*.py      # 0001_initial.py 제외
rm simulation/migrations/*.py   # 0001_initial.py 제외

# 3. DB 테이블 초기화 (주의: 모든 데이터 삭제!)
python manage.py migrate input_data zero
python manage.py migrate common zero
python manage.py migrate simulation zero

# 4. 재마이그레이션
python manage.py migrate
```

## **8. 운영 모니터링 (Operations & Monitoring)**

### **8.1 로그 위치 및 분석**

```
logs/ (프로젝트 루트)
├── django.log          # Django 애플리케이션 로그
├── celery.log          # Celery 워커 로그
├── database.log        # PostgreSQL 쿼리 로그
└── access.log          # HTTP 요청/응답 로그

# 실시간 로그 보기
tail -f logs/django.log

# 에러 로그만 필터링
grep ERROR logs/django.log
grep "Traceback" logs/django.log
```

### **8.2 성능 모니터링**

```bash
# 1. 느린 쿼리 식별
# Django Debug Toolbar (DEBUG=True일 때)
```

```bash

# 2. 백업 복구

# 3. Docker 환경에서 백업
docker-compose exec db pg_dump -U postgres cascading_opt > backup.sql

# 4. Docker 환경에서 복구
docker-compose exec -T db psql -U postgres cascading_opt < backup.sql
```

## **10. 트러블슈팅 (Troubleshooting)**

| 증상 | 원인 | 해결 방법 |
|------|------|---------|
| `ModuleNotFoundError: No module named 'django'` | 가상환경 미활성화 또는 패키지 미설치 | `pip install -r requirements.txt` 재실행 |
| `ProgrammingError: relation "..." does not exist` | 마이그레이션 미실행 | `python manage.py migrate` 실행 |
| `FATAL: Ident authentication failed for user "postgres"` | PostgreSQL 연결 실패 | `DATABASE_URL` 확인, PostgreSQL 서비스 시작 |
| `ConnectionRefusedError: [Errno 111] Connection refused` | Redis 미실행 | `docker run ... redis:7` 또는 `redis-server` 실행 |
| `FileNotFoundError: [Errno 2] No such file or directory: 'input_data/data/base_data/*.csv'` | CSV 파일 경로 오류 | 파일 존재 확인, 절대경로 사용 |
| `IntegrityError: duplicate key value violates unique constraint` | 중복 데이터 적재 | `init_base_data`에 `--force` 옵션 추가 |
| `UnicodeDecodeError: 'utf-8' codec can't decode byte` | CSV 파일 인코딩 오류 | UTF-8 with BOM 형식으로 재저장 |
| `Celery Worker 실행 안 됨` | Redis 미연결 | `CELERY_BROKER_URL` 확인, Redis 실행 상태 확인 |
