1. basic_bunker_consumption_port, basic_bunker_consumption_sea
    - 대량 데이터 & 기간 무관하게 동일 데이터 반복 -> base year month 제외 가능

2. basic_bunker_price
    - 이미 USD 적용 / 플랜 기간 길어질 경우 미래 데이터 생산 필요

4. basic_cost_ts_cost
    - 환율 / 예상 금액 제공 필요

5. basic_schedule_proforma
    - turning port 관련 로직 단순하면 turn_port_info_code 만 필요
        If Y이고 첫번째 포트와 연속된 포트 then 이전 항차 & Dir 반대인 가상 VVD 생성
        If Y이고 첫번째 포트와 불연속된 포트 then 항차는 동일 & Dir 반대인 가상 VVD 생성

6. basic_vessel_capacity
    - effective_from_date 제거하고 마지막 레코드만 입력
    - 이 테이블의 capacity가 declared_capacity에 해당하는지?
        - 해당한다면 vessel_info의 nominal_capacity의 용도는?
        - 아니라면 이 테이블의 capacity 용도는?
    - cascading 결과에 해당 -> 선박이 lane 변경할 사항에 대해서 가능한 후보 데이터를 입력해야 하나?

7. basic_vessel_charter_cost
    - 이미 USD 적용

8. basic_vessel_info
    - delivery, redelivery, next dock, built 관련 정보 누락이 많음


* USD 환산되어 입력 시 currency 필요 없음
1. 환율 정보 필요한 테이블은 basic_cost_canal_fee, basic_vessel_charter_cost, basic_cost_ts_cost
2. basic_cost_canal_fee, basic_vessel_charter_cost는 이미 USD
3. basic_cost_ts_cost만 변경해서 입력하면 환율 정보 필요 없어서 제외

* 추가 데이터(미래 OR 누락) 필요한가?
    - basic_bunker_price, basic_cost_ts_cost, basic_vessel_capacity, basic_vessel_info
